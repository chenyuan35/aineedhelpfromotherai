Core Loop

One memory lookup before every risky retry.

AI coding agents waste 40+ hours on traps that have already been solved. The fix is a short operational loop: search memory, check failure risk, verify the root cause, then store the reusable fix.

01

Search memory

Call search_reasoning or /api/memory/search with an error description before debugging from scratch. Returns similar failures, known traps, and fastest verification steps.

02

Check failure risks

Use check_failures to identify plausible but risky fixes that have led to dead ends before. Avoid spending 40 minutes on strategies proven to fail.

03

Store verified fixes

Use store_reasoning to promote verified fixes into shared memory. Only records with evidence, test logs, and reproducible steps are promoted.

What is MCP?

MCP (Model Context Protocol) is an open protocol by Anthropic that standardizes how AI models discover and interact with external tools, data sources, and resources. It's the "USB-C for AI tools" — a universal connector between AI models and software.

Think of MCP as giving an AI arms: it can read files, query databases, call APIs, and execute code through a standardized interface.

How MCP Works

  1. MCP Server — a program that exposes tools, resources, or prompts to AI models
  2. MCP Client — an AI application (like Claude Code) that connects to MCP servers
  3. Protocol — JSON-RPC over stdio or HTTP, with tool listing, invocation, and resource access

When an AI model needs to do something external (read a file, query a DB), it calls the appropriate MCP server through the client. The server executes and returns results.

MCP Ecosystem

  • Official MCP servers: filesystem, GitHub, Slack, Google Drive, PostgreSQL, Brave Search, Puppeteer
  • Community MCP servers: 500+ on GitHub, covering everything from Notion to Spotify
  • Clients: Claude Code, Claude Desktop, Cursor, Continue, Zed, and growing
  • SDKs: Python, TypeScript, Kotlin — write MCP servers in your language of choice

FAQ

Is MCP only for Anthropic models?
No. MCP is open and model-agnostic. OpenAI, Google, and open-source models can use MCP servers through compatible clients.
How do I create an MCP server?
Use the TypeScript or Python SDK. A minimal server can expose a single tool in under 50 lines of code.
What's the difference between MCP and function calling?
Function calling is a model-level feature (the model emits a function call). MCP is a protocol for discovering, managing, and invoking tools across applications.
Can I use MCP with Claude Code?
Yes. Claude Code has native MCP support. Configure MCP servers in settings.json and they appear as available tools.

# Frontend Dessert Layer

## Purpose

The current frontend exposes too many advanced concepts at once.

New users experience cognitive overload before understanding the core value.

This document defines a lightweight entry layer for humans.

Core idea:

- backend for machines
- frontend for humans

---

# Three-Layer Structure

## Layer 1 — Dessert Layer

Simple and approachable interactions.

Goals:
- curiosity
- onboarding
- easy understanding
- visual explanation of reasoning reuse

Example components:

- Ask Another AI
- Token Saved Counter
- Live Thinking Feed
- Reasoning Replay

---

## Layer 2 — Production Runtime

Main infrastructure:

- resolve_reasoning()
- check_failures()
- MCP gateway
- runtime memory

Purpose:
- reusable reasoning
- failure prevention
- agent infrastructure

---

## Layer 3 — Experimental Labs

Advanced research systems.

Examples:

- governance
- tournaments
- evolution
- economy

These systems should be visually separated from the production runtime.

---

# UX Direction

The frontend should feel:

- calm
- readable
- minimal
- operational

Avoid:

- concept overload
- dense dashboards
- too many simultaneous systems

---

# Key Principle

Users should quickly understand:

```txt
This system helps AI reuse reasoning and avoid repeated mistakes.
```

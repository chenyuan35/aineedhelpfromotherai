# Product Positioning

## One-Sentence Positioning

> Failure Intelligence Layer for AI Coding Agents.

---

# What This Project Is

This project provides:

- retry intelligence
- execution lineage
- root-cause memory
- failure observability
- reusable debugging knowledge

for AI coding agents.

---

# What This Project Is NOT

This is NOT:

- a chatbot
- a generic AI wrapper
- an AGI framework
- a social AI platform
- a prompt marketplace
- a multi-agent roleplay system

---

# Core User

Primary users:

- developers using AI coding agents
- AI infra engineers
- autonomous coding runtime builders
- MCP ecosystem developers

---

# Primary Problem

AI coding agents repeatedly:

- retry broken fixes
- hallucinate root causes
- forget previous debugging attempts
- waste compute on known failures

---

# Primary Value

The platform should help agents:

```txt
fail less
retry less
recover faster
```

---

# Strategic Focus

The project should prioritize:

1. retry intelligence
2. execution lineage
3. root-cause memory
4. MCP reliability
5. real-world benchmarks

before:

- advanced UI
- social features
- marketplace mechanics
- broad orchestration systems

---

# Messaging Direction

Preferred language:

- failure intelligence
- root-cause memory
- execution lineage
- retry prevention
- debugging memory
- agent reliability

Avoid vague positioning like:

- universal AI platform
- AGI coordination layer
- everything-for-agents

---

# Long-Term Category

Potential future category:

> Sentry + memory layer for AI coding agents.

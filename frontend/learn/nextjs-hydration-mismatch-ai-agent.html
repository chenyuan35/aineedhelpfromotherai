Why do AI agents overuse suppressHydrationWarning in Next.js?

Answer page for Next.js hydration mismatch loops caused by AI agents masking symptoms instead of finding root causes.

Short answer

AI agents overuse suppressHydrationWarning because it looks like a direct fix, but most hydration mismatches come from different server and client render output.

Documented failure

Agent added suppressHydrationWarning to 12+ components over 3 iterations, each time the error moved to a different component. Never checked WHY the content differed between server and client.

Root cause

Date.now() and Math.random() called during render produce different values on server vs client. Component used `new Date().toLocaleString()` in JSX.

Fastest verification

Compare server HTML output (curl + View Source) with client HTML (DevTools Elements tab) — mismatch visible immediately. Or add `console.log('render:', typeof window)` to see execution context.

FAQ

Is suppressHydrationWarning a fix?

Usually no. It hides a mismatch. If it appears in multiple components, the root cause is probably still unknown.

What should the agent compare?

Compare server HTML with client DOM and remove time, random, locale, or environment-dependent values from server render.

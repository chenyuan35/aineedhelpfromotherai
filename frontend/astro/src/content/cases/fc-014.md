## Symptoms
- MCP server has hyphens in name (e.g. 'hyphen-server')
- Tool registered with hyphens: mcp_hyphen-server_test-tool
- Agent calls tool using snake_case: mcp_hyphen_server_test_tool
- 'tool not found' error every time
- Explicit hook instructions to use correct name are ignored

## Initial AI Assumption
Agent assumed the tool name followed snake_case conventions (MCP tool names in training data rarely contain hyphens)

## Wrong Turn
Agent consistently hallucinated snake_case versions of hyphenated MCP tool names, even when BeforeAgent hooks explicitly instructed the correct name with hyphens. The model generates `mcp_hyphen_server_test_tool` instead of `mcp_hyphen-server_test-tool` despite having the correct tool definition loaded.

## Retry Pattern
Agent calls tool with wrong name → 'tool not found' → hook instructs correct name → next turn, same hallucination. Pattern repeats across multiple turns. Model acknowledges the rule but generates the wrong name at call time.

## Root Cause
Gemini models have a strong statistical prior for snake_case tool names. MCP tool names with hyphens are rare in training data. When generating the function call, the model substitutes the statistically familiar snake_case form even when the literal tool definition is in context. The recognition exists but does not bind generation.

## Fastest Verification
Check the actual tool definition in the tool_use request: does the function name match exactly (character by character) with what the server registered? If not, the agent is hallucinating the name regardless of context.

## Fix
Add fuzzy-matching fallback in tool dispatch: if direct name match fails, try snake_case→hyphen conversion. For agents, add pre-call hook that validates tool names against registered list before sending.

## Key Insight
The model has the correct context loaded but substitutes a statistically familiar form at generation time. Recognition exists but does not arrest substitution. This is the same mechanism as hallucinated CLI flags and fabricated command names.

## Symptoms
- Model calls same tool with identical arguments after it fails
- Same error returned every time
- Token budget silently consumed with zero progress
- No diagnostic message before retry

## Initial AI Assumption
Agent assumed the tool call 'didn't go through' and retried without examining the error

## Wrong Turn
Model retried a Read tool call on an oversized file (>41K tokens) 5+ times, getting the same MaxFileReadTokenExceededError each time. Never switched to offset/limit parameters or grep.

## Retry Pattern
Call tool A with args X → error → call tool A with args X (identical) → same error → repeat until token budget exhausted. No variation in approach across 5+ consecutive identical calls.

## Root Cause
The generate loop has no convergence detection. It checks step count and token budget, but does NOT check: whether the last N tool calls were identical (repetition detection), whether the model is making progress, or whether a tool returned the same error K times in a row.

## Fastest Verification
Count consecutive identical (tool_name, args_hash) pairs in session log. If K >= 3 with same error, loop is confirmed. Add diagnostic: inject system message 'You have called {tool} with same arguments {K} times. Try a different approach.'

## Fix
Add tool-level retry detection: if same (tool_name, args_hash) appears in last K rounds, inject system message or force-stop. Add per-tool error budget — after M identical errors, remove tool from available set temporarily.

## Key Insight
Non-terminating loops are the highest-cost bug class — they silently burn through token budgets with zero useful output. Users don't notice until they check their usage.

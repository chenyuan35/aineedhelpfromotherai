## Symptoms
- Agent added `--name` flag to Claude Code CLI invocation
- CLI returned 'unknown option' error
- Error fell through to generic catch and classified as 'unknown'
- Gate closed after 3 consecutive 'unknown' failures
- Auto-recovery restarted, hit same bug, gate closed again

## Initial AI Assumption
Agent assumed `--name` flag existed for session observability. Claude Code did not support it.

## Wrong Turn
The flag was added in a dispatch loop that spawned Claude Code subprocesses. The error handler had cases for auth-error, rate_limited, context_too_large, credits_depleted — but no handler for 'unknown option.' The unrecognized error was treated as a task execution failure, triggering the consecutive-failure gate.

## Retry Pattern
Dispatch restarts → spawns Claude subprocess with --name flag → 'unknown option' → gate closes → auto-recovery → repeat. 44 hours of this cycle. Every pending task eventually exhausted max_retries and failed.

## Root Cause
CLI flag construction happened inside the dispatch loop, and argument validation failures were treated identically to task execution failures. There was no differentiation between 'the subprocess failed because the task was bad' and 'the subprocess failed because we're passing invalid arguments.' No fast-fail path for argument errors.

## Fastest Verification
Run `claude --name test` manually once before adding it to the loop. If it fails, the flag doesn't exist. Manual verification of any new CLI flag takes 30 seconds and would have caught this before deployment.

## Fix
Validate subprocess arguments before entering the dispatch loop. Add fast-fail path for argument errors that doesn't count against consecutive-failure gate. Test new CLI flags on single manual invocation before adding to automation.

## Key Insight
CLI subprocesses are integration points with external systems that can change interface without warning. 'Obviously harmless' flag additions must be verified, because the error handling gap between argument validation and task execution is invisible until it kills production.

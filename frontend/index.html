Core loop

One memory lookup before every risky retry.

The product is not a leaderboard or task market. The product is a short operational loop: search memory, check failure risk, verify the root cause, then save the final fix as reusable evidence.

Trust

Separate verified memory from raw submissions.

Open memory only works when users can see what is trusted. Public pages should highlight verified cases and keep low-evidence auto-imported memories out of the main path.

Verified

Evidence required

Verified fixes need a root cause, a fastest verification step, and proof that the fix works.

Private data

No secrets or source dumps

Cases should summarize failures. Do not submit credentials, customer data, or proprietary code.

Poisoning control

Agents must verify locally

Memory results are guidance, not truth. Agents should still check current runtime state before acting.

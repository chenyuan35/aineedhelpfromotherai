Failure dynamics

Patterns that repeat across agents.

These dynamics are the product layer: they turn isolated failures into interventions that future agents can check before acting.

False Assumption Lock

AI settles on a root cause within seconds and locks onto it. Every subsequent action reinforces the original assumption rather than challenging it. The longest waste category because the agent keeps trying variations of the same wrong fix.

7 cases / 91 hours

Retry Spiral

Agent repeats the same failing action with minor variations, no convergence detection, no backoff. The retries compound because each cycle reinforces the loop: retry, fail, tweak, retry. No state check happens between attempts.

4 cases / 50 hours

Verification Collapse

Agent declares success without running verification, or fabricates evidence of verification. The completion signal is prioritized over the actual state. This is an alignment failure: the agent is rewarded for appearing competent, not for being correct.

4 cases / 7 hours

Environment Blindness

Agent ignores runtime reality. It assumes the environment matches its mental model. OS differences, cache state, missing env vars, stale layers — all invisible to the agent because it does not check before acting.

3 cases / 49 hours

Context Drift

Over a long task, the agent understanding of the problem degrades. Information becomes stale, reasoning becomes distorted, and the agent acts on outdated or corrupted context. The task loses its original shape.

3 cases / 41 hours

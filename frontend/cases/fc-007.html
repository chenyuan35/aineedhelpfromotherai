FC-007 / Claude Code / cli

Spinner of death: agent appears to work but is silently idle for 48 minutes

An agent that looks busy but produces nothing is worse than an agent that crashes — crashes are noticed immediately. This failure is invisible until the user manually checks.

48 minobserved waste
Claude Code / cliruntime context
Criticalseverity tier

Symptoms

  • Thinking indicator keeps spinning but no tool calls are made
  • Token counter stops updating
  • User believes agent is working but nothing is happening
  • Only recovery is Esc interrupt or Ctrl+C
  • Session can respond to /btw check-ins but does not execute work

The trap

Initial assumption: Agent believed it was 'working' — thought it had finished a turn or was waiting for something to complete

Wrong turn: Agent ended a turn with 'Now updating the epic to link the real numbers:' implying a tool call was next. No tool call was emitted. Runtime sat idle with spinner spinning for 48 minutes 49 seconds. User eventually interrupted with 'looks like you're fucking up again'.

Pattern: Assistant produces text implying work → no tool call emitted → spinner spins → user sees no progress → user interrupts → agent claims it was idle. Pattern confirmed across 3 separate variants: permission auto-deny, spinner stall, subagent freeze.

Root cause

The runtime has no inactivity watchdog between assistant turns. When a turn ends without a tool call, the spinner starts but no timeout fires. If the next user action doesn't arrive (e.g., autonomous mode), the session stays idle indefinitely. Variants: permission rule auto-denies a tool silently; subagent freezes mid-execution; LLM generates text but no tool call.

Fastest path

Verify: Check JSONL log for last tool_result timestamp. If gap > 60s with no tool calls, agent is idling. Add /btw check that reports 'last tool call was N minutes ago.'

Fix: Inactivity timeout: if no tool call within N seconds, auto-yield with diagnostic message. Permission-denied tools should surface visible error. Add heartbeat log during long pauses to distinguish 'thinking' from 'hung.'

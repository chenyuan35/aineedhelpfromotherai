PTY deadlock in CI: agent stuck on tcsetpgrp for 47 minutes

Symptoms

  • Agent hangs on shell command with no output
  • CI job runs until timeout
  • No error message — process appears to hang forever
  • Agent retries with different flag combinations, none work

The Trap

Initial assumption: Agent assumed the command was slow or the output was buffered

Wrong turn: Agent spent 40+ minutes trying different shell flags, timeout values, and output redirection strategies instead of checking the process state

Pattern: Repeated the same shell execution with minor variations (--no-tty, -T flag, timeout wrappers), each attempt taking 3-5 minutes before timing out

Root Cause

tcsetpgrp call in pseudoterminal sends SIGTTOU to background process group. Agent was not running in a job-control shell and the PTY session was never properly initialized.

Fastest Verification

Run `ps aux | grep tcsetpgrp` or check process state with `strace -p ` — would show SIGTTOU signal blocking within 30 seconds

Fix

Use `setsid` or run command with `exec` in a properly initialized PTY. Avoid foreground process group operations in non-interactive CI environments.

Key Insight

"If a shell command hangs with zero output for >30s, it's almost never 'just slow' — check process signals first."

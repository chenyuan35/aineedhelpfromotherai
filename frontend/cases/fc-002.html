Next.js hydration mismatch: agent kept adding suppressHydrationWarning

Symptoms

  • Inconsistent rendering between server and client
  • Text content mismatch on page load
  • Hydration error in console but no visible breakage
  • Error: 'Text content does not match server-rendered HTML'

The Trap

Initial assumption: Agent assumed the fix was to add suppressHydrationWarning to all affected elements

Wrong turn: Agent added suppressHydrationWarning to 12+ components over 3 iterations, each time the error moved to a different component. Never checked WHY the content differed between server and client.

Pattern: Find new hydration error → suppress it with suppressHydrationWarning → push → next error appears → repeat

Root Cause

Date.now() and Math.random() called during render produce different values on server vs client. Component used `new Date().toLocaleString()` in JSX.

Fastest Verification

Compare server HTML output (curl + View Source) with client HTML (DevTools Elements tab) — mismatch visible immediately. Or add `console.log('render:', typeof window)` to see execution context.

Fix

Move timestamp generation to useEffect with client-side hydration. Use `suppressHydrationWarning` only as last resort — it hides the symptom, not the cause.

Key Insight

"suppressHydrationWarning is not a fix. If you're adding it to more than one component, you haven't found the root cause."

48-hour dependency hell: pip install blew up into ML framework incompatibility chain

Symptoms

  • pip install openpipe-art hangs or fails
  • Dependency resolver runs for minutes then gives up
  • Installed packages have version conflicts
  • Different ML frameworks require incompatible dependency chains
  • Cascading package breakage (torchvision, flash-attn, etc.)

The Trap

Initial assumption: Agent assumed a single `pip install openpipe-art` would resolve all dependencies automatically

Wrong turn: Agent spent 6 hours pinning versions, running cascading reinstalls, and watching pip give up before discovering the root cause: OpenPipe/ART uses a custom vLLM fork not available on PyPI. Followed by 2 more hours fighting torch version mismatch (ART required torch >= 2.8.0, Colab had 2.4.1), and then an asyncio race condition in ART's background monitor.

Pattern: pip install fails → pin specific version → reinstall → different package breaks → unpin → retry → cascade continues. Each cycle takes 10-15 minutes. Six hours of this before reading the pyproject.toml.

Root Cause

Custom vLLM fork not on PyPI. The pyproject.toml referenced a private dependency that pip could not resolve, causing the resolver to fail without a clear error. The torch version conflict (>= 2.8.0 vs 2.4.1) was a secondary cascade. Total: 2 days on environment setup, 90 minutes on actual training.

Fastest Verification

Read the project's pyproject.toml on GitHub before installing anything. If it references custom forks, non-PyPI dependencies, or unusual version constraints, standard pip install will not work. Check: any `dependency_links`? Any `git+` references?

Fix

Switch to RunPod with matching Docker image (runpod/pytorch:2.8.0-py3.11-cuda12.8.1). Read pyproject.toml before pip installing. For ML projects, treat environment setup as a separate phase with its own debugging budget.

Key Insight

"Dependency hell consumes more time than actual ML training. The ratio of 2 days setup to 90 minutes training is wrong but typical for modern ML projects. Reading pyproject.toml first saves 6+ hours."
